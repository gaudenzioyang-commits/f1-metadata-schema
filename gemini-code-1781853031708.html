a) Opening Narrative

도메인 내 메타데이터 역사, 타겟 유저, 개발 노하우 및 이론적 배경

1. 모터스포츠 도메인 메타데이터의 역사와 배경

포뮬러 원(Formula 1)은 초창기 피트 보드(Pit boards)와 수동 스톱워치를 이용한 아날로그 기록 측정 방식에서 시작하여, 현대에는 차량 한 대당 수백 개의 센서가 실시간 스트리밍 데이터를 뿜어내는 초정밀 데이터 집약형 스포츠로 진화했습니다. 그러나 이러한 방대한 양의 랩 타임, 피트 스톱, 타이어 마모도 데이터는 각 컨스트럭터(팀)나 주관사(FIA, 자회사 등)의 폐쇄적인 포맷이나 단순 플랫 CSV 데이터셋 형태로 파편화되어 존재해 왔습니다. 이로 인해 정보학적 관점에서의 정보 공유, 장기 보존, 크로스-레이스 분석(Cross-race analysis) 측면에서 상호운용성(Interoperability) 보장에 큰 기술적 한계가 존재했습니다.

본 메타데이터 스키마는 파편화된 F1 레이스 결과와 페이스 분석 데이터셋을 의미적으로 연결하고 체계적으로 기술하기 위해 고안되었습니다.

2. 상정 유저 (Intended Users)

본 스키마의 주요 타겟 이용자는 다음과 같습니다:
• 스포츠 데이터 분석가 및 저널리스트: 경기 간 타이어 전략과 페이스 변화를 기계 가독형 데이터로 신속히 추출 및 비교 분석하고자 하는 전문가 그룹.
• 모터스포츠 아카이비스트 및 역사학자: 수십 년간 축적된 F1의 역사적 레이스 정보와 원시 데이터셋의 출처 및 맥락을 장기 보존하려는 큐레이터.
• F1 오픈데이터 개발자 및 데이터 취미 분석가: 파이썬(Python)이나 R을 활용하여 오픈 텔레메트리 데이터 분석 플랫폼을 구축하려는 에코시스템 기여층.

3. 개발 노트 및 이론적 고찰 (평면 구조에서 계층 구조로의 진화)

초기 드래프트 3 설계에서는 관계형 데이터베이스의 테이블 뷰와 유사하게 모든 엘리먼트가 수평적으로 나열된 평면형(Flat) 구조를 취했습니다. 그러나 이러한 구조는 실제 그랑프리 레이스의 현실을 반영하지 못한다는 동료 및 TA의 결정적인 피드백을 받았습니다. 실제 레이스에서 하나의 레이스 자원 밑에는 20명의 드라이버 기록이 수반되며, 각 드라이버는 경기 중 타이어를 여러 번 바꾸는 '다중 타이어 스틴트(Multiple Tire Stints)' 과정을 거치기 때문입니다.

이를 해결하기 위해 본 최종 스키마에서는 [계층형 중첩 구조(Nested Hierarchical Structure)]를 명확히 도입하였습니다. f1:DriverRecord라는 상위 래퍼 엘리먼트를 두어 특정 드라이버명, 소속 팀, 최종 순위를 유기적으로 결합시켰으며, 그 하위에 f1:TireStints와 반복 가능한(Repeatable) f1:Stint 엘리먼트를 다중 선언할 수 있도록 보완하여 모터스포츠의 도메인 특성을 완벽히 추상화하는 데 성공했습니다.

b) Relevant Metadata Standards

본 스키마 구현에 차용 및 융합된 외부 글로벌 메타데이터 표준

1. Dublin Core Metadata Terms (dcterms / dc)

선정 이유: 정보 자원의 발견과 식별을 위한 글로벌 범용 표준입니다. 본 스키마에서는 자원의 핵심 식별자인 타이틀(dcterms:title), 생성일자(dcterms:date), 설명(dcterms:description), 주체(dcterms:creator), 고유 식별코드(dcterms:identifier) 등을 기술하기 위해 도입하였습니다. 이를 통해 모터스포츠 데이터가 다른 문화예술 및 학술 아카이브 인프라와 표준 규격으로 연계될 수 있는 상호운용성의 교두보를 마련했습니다.

2. Data Catalog Vocabulary (dcat)

선정 이유: F1 레이스 원시 원격 측정 및 결과 기록은 본질적으로 웹상에서 유통되는 '데이터셋(Dataset)'입니다. 따라서 해당 자원이 어떤 포맷(CSV, JSON, XML)으로 제공되고 유통되는지 데이터 카탈로그 관점에서 정의하기 위해 W3C 표준인 DCAT의 dcat:distribution 모델 개념을 수용하여 데이터 자원의 배포 속성을 보장했습니다.

3. Schema.org (schema) 및 Custom Namespace (f1)

선정 이유: 경기 대회(Event), 인물(Person), 조직(Organization), 장소(Place) 간의 관계형 개념 모델 구조를 아키텍처 관점에서 차용하기 위해 수용했습니다. 최종 검증 단계에서는 완전한 모터스포츠 고유 제약 조건을 만족시키기 위해 도메인 전용 독자 네임스페이스(f1:)를 수립하고, 하위에 완벽한 통제어휘집 기반의 엘리먼트들을 독자 선언하여 정밀도를 극대화했습니다.

2. Controlled Vocabularies Appendix

• Race Position Vocabulary: 1부터 20까지의 양의 정수 또는 레이스 미완주/실격 코드로 제한함: [1-20], DNF(Did Not Finish), DNS(Did Not Start), DSQ(Disqualified).
• Tire Compound Vocabulary: 피라미드 규격에 부합하는 피렐리 공식 지정 5대 컴파운드로 강제 제한함: Soft, Medium, Hard, Intermediate, Wet.

d) XML Schema Definition (XSD) File

본 메타데이터 표준을 강제하는 주석 및 문서화 내장형 완벽 유효 스키마

f1_schema.xsd 
<?xml version="1.0" encoding="UTF-8"?>
<!-- 
  ========================================================================
  [PROJECT] Formula 1 Race Results and Race Pace Metadata Schema (V4)
  [AUTHOR] Dept. of Library and Information Science, Yang Seung-hyun (2025310382)
  [DESIGN INTENT COMMENTARY]
  - Root Node: <RaceRecord> encompasses both global race-level contexts and 
    nested individual achievements.
  - Namespace Architecture: Explicitly imports Dublin Core Terms via 'ref' to ensure 
    global catalog validation interoperability.
  - Hierarchical Fix: Per peer/TA feedback, a flat structure is abandoned. 
    <DriverRecord> now encapsulates interconnected attributes (Driver, Team, Position) 
    and handles multiple repeatable <Stint> tire tags for compound progression mapping.
  ========================================================================
-->
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"
           targetNamespace="http://example.org/f1/schema/"
           xmlns:f1="http://example.org/f1/schema/"
           xmlns:dcterms="http://purl.org/dc/terms/"
           elementFormDefault="qualified">

    <!-- Import Official Dublin Core Metadata Terms Standard Schema -->
    <xs:import namespace="http://purl.org/dc/terms/" 
               schemaLocation="https://www.dublincore.org/schemas/xmls/qdc/dcterms.xsd"/>

    <!-- ROOT ELEMENT DECLARATION -->
    <xs:element name="RaceRecord">
        <xs:annotation>
            <xs:documentation>The root container holding global Grand Prix event metadata and the nested collection of driver metrics.</xs:documentation>
        </xs:annotation>
        <xs:complexType>
            <xs:sequence>
                <!-- Global Descriptors via Dublin Core -->
                <xs:element ref="dcterms:title" minOccurs="1" maxOccurs="1"/>
                <xs:element ref="dcterms:date" minOccurs="1" maxOccurs="1"/>
                <xs:element ref="dcterms:description" minOccurs="0" maxOccurs="1"/>
                <xs:element ref="dcterms:creator" minOccurs="1" maxOccurs="1"/>
                <xs:element ref="dcterms:identifier" minOccurs="1" maxOccurs="1"/>
                
                <!-- Domain-Specific High-Level Data -->
                <xs:element name="GrandPrixName" type="xs:string" minOccurs="1" maxOccurs="1"/>
                <xs:element name="Season" type="f1:seasonYearType" minOccurs="1" maxOccurs="1"/>
                <xs:element name="CircuitName" type="xs:string" minOccurs="1" maxOccurs="1"/>
                
                <!-- Nested Driver Blocks: Solves Flat-Structure Integrity Flaws -->
                <xs:element name="DriverRecord" minOccurs="1" maxOccurs="unbounded">
                    <xs:complexType>
                        <xs:sequence>
                            <xs:element name="DriverName" type="xs:string" minOccurs="1" maxOccurs="1"/>
                            <xs:element name="TeamName" type="xs:string" minOccurs="1" maxOccurs="1"/>
                            <xs:element name="RacePosition" type="f1:racePositionType" minOccurs="1" maxOccurs="1"/>
                            
                            <!-- Multi-Stint Progression Container -->
                            <xs:element name="TireStints" minOccurs="0" maxOccurs="1">
                                <xs:complexType>
                                    <xs:sequence>
                                        <!-- Repeatable Stint Order mapping real race compound shifts -->
                                        <xs:element name="Stint" type="f1:tireCompoundType" minOccurs="1" maxOccurs="unbounded"/>
                                    </xs:sequence>
                                </xs:complexType>
                            </xs:element>
                        </xs:sequence>
                    </xs:complexType>
                </xs:element>
            </xs:sequence>
        </xs:complexType>
    </xs:element>

    <!-- CONTROLLED VOCABULARY SIMPLE TYPES -->
    <xs:simpleType name="seasonYearType">
        <xs:restriction base="xs:integer">
            <xs:minInclusive value="1950"/>
            <xs:maxInclusive value="2099"/>
        </xs:restriction>
    </xs:simpleType>

    <xs:simpleType name="racePositionType">
        <xs:restriction base="xs:string">
            <xs:pattern value="([1-9]|1[0-9]|20)|DNF|DNS|DSQ"/>
        </xs:restriction>
    </xs:simpleType>

    <xs:simpleType name="tireCompoundType">
        <xs:restriction base="xs:string">
            <xs:enumeration value="Soft"/>
            <xs:enumeration value="Medium"/>
            <xs:enumeration value="Hard"/>
            <xs:enumeration value="Intermediate"/>
            <xs:enumeration value="Wet"/>
        </xs:restriction>
    </xs:simpleType>

</xs:schema>
